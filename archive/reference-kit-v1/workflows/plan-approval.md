# 計画承認workflow

## 使う場面

実装、削除、ファイル生成、インストール、外部反映を始める前に使います。

## 手順

1. 調査・読取で既存構成を確認する。
2. 目的、範囲、影響、検証、停止条件、不明点、懸念点、代替提案、ユーザー判断が必要なリスクを含む計画を作る。
3. ユーザーへ提示し、承認を得る。
4. 承認済みplanは `.agentops/plans/current.md` に置く。
5. 今回セッションの実行順は `.agentops/task-plans/current.md` に置く。
6. 未完了の子taskは `.agentops/tasks/*.md` に置く。
7. 完了したtaskやplanは `.agentops/archive/<plan-id>/` へ移す。

## 承認省略時

承認済みの同一planをセッション跨ぎで継続する場合だけ承認を省略できます。ただし、今回セッションの小分け計画とtaskは必ず提示します。
