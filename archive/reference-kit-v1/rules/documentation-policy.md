# ドキュメント更新ルール

## ルール

実装差分がREADME、docs、設定、prompt、skill、workflow、運用手順に影響する場合は、ドキュメント更新を完了条件に含めます。

実プロジェクトでは、仕様、API、環境変数、デプロイ手順、rollback、監視、リリースノート、顧客向け説明に影響があるかも確認します。

変更したルールやworkflowが実設定へ反映される必要がある場合は、`config/` の雛形も更新します。

## 記録

セッションをまたぐ場合は、`.agentops/handoffs/` または `.agentops/prompts/` に次の作業者が復帰できる情報を残します。
