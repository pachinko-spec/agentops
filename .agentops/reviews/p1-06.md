## Codex 所見

### P0
- なし

### P1
1. **plan-id パストラバーサル**: `PLAN_ID_PATTERN` が `/` と `.` を許可し、`detect_active_plan_id()` 後に `_validate_plan_id()` を通していない。`> plan-id: \`../../outside-plan\`` で `.agentops/archive` 外へ task を移動できることを一時 project で確認済み。`PLAN_ID_PATTERN` 自体を狭めるとともに、戻り値も検証すべき。
2. **メタデータ更新と move の順序不整合**: `archive_task` は `next-session.md` 更新後に `move_path` を呼ぶ。移動先で `FileExistsError` などが発生すると task は元に残ったまま `next-session.md` だけ完了扱いになる。`archive_plan` も README row 挿入が move より先で同種の不整合リスク。preflight + move 成功後にメタデータ更新へ寄せ、`OSError` を `AgentOpsError` で包むべき。

### P2
1. **`completed_tasks: []` inline 形式非対応**: `completed_tasks` パースはブロック形式だけ。inline 空配列で CLI は `completed_tasks += 01-demo` と表示する一方、実ファイルには追記されない。docs の「配列に追記」契約として不安定。inline 空配列をブロック形式へ正規化するか、非対応形式なら exit 2 とする。
2. **CLAUDE.md / AGENTS.md と実装・docs の文言ズレ**: durable instructions では「entry_point を空欄に書き換える」と書いたが、実装と docs/11 はマーカー文字列を書く。文言を実装側（マーカー文字列）に揃える。

### P3
1. **docs/11 の `..` 制約と SAFE_PLAN_ID_PATTERN の不一致**: docs/11 は「`..` を含まない前提」と書いたが、`SAFE_PLAN_ID_PATTERN` は `a..b` を許可する。slash 禁止のため traversal にはならないが、仕様と実装を揃えるべき。
2. **`--date` の YYYY-MM-DD 検証と `--summary` の table 文字 escaping 不足**: archive README の table を壊し得るので hardening。
