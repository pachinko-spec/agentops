task 08 §不変条件「CI と hook の責務分担」は、本セッションでは以下の意味で運用する:

- CI (`.github/workflows/ci.yml`): GitHub 側の強制 gate。fail 系 3 job (actionlint / yamllint / markdown-link-check) は PR を block する。warn 系 1 job (freshness-check) は job summary 出力のみで block しない。

- 二重責務なし: hook は branch/test の commit/push gate、CI は yaml/markdown/link/freshness の PR gate。検証対象が完全に分離されている。

4. **markdown-link-check の rate limit 対策**: `retryOn429: true` / `retryCount: 3` / `fallbackRetryDelay: "30s"` / `timeout: "20s"` / `aliveStatusCodes: [200, 206, 999]` / `ignorePatterns` (localhost / example.com / GitHub issues / PRs) が GitHub の rate limit + 外部 URL 一時失敗を捌けるか。false positive 頻度が運用に耐えるか。
