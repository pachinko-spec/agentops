# handoffs

**plan で考えた task の範囲を超えた持ち越し** を残す場所です。

具体的には次のような内容を扱います。

- 別 plan への申し送り
- 長期 blocker（環境依存、外部要因、未解決の設計判断など）
- 計画外で発生した観察事項

PR 単位の進捗（在庫管理、状態同期、次セッションで続ける作業）には使いません。それは `.agentops/tasks/*.md` の責務です。

## 完了 handoff の扱い

完了した plan に紐づく handoff は、対応する `.agentops/archive/<plan-id>/handoffs/` へ移します。`.agentops/handoffs/` 直下は **進行中のハンドオフだけ** にします。

## 必須項目

次に読むエージェントが、短い文脈で再開できることを優先します。

- 今回完了したこと
- 未完了のこと
- 現在のブランチ
- 変更ファイル
- 実行したテスト
- 未解決リスク
- 次セッションへ投入するプロンプト

この README は他プロジェクトの `.agentops/handoffs/` へコピーして使うことを前提に self-contained にしています。コピー先プロジェクトに本リポジトリの `docs/02-workflow.md` は存在しないため、必須項目はここで完結させます。
